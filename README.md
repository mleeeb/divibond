# ДивиБонд — Telegram Mini App

MVP мини-приложения для просмотра облигаций и дивидендных акций MOEX с
фильтрацией по доходности, цене и сроку до погашения. Интерфейс — на русском,
в стиле банковского приложения (светло-сиреневый + тёмно-синий).

## Структура проекта

```
divibond/
├── backend/
│   ├── app/
│   │   ├── main.py         — FastAPI-приложение (HTTP-эндпоинты)
│   │   ├── data.py          — сборка данных: облигации/акции (без FastAPI)
│   │   ├── moex_client.py    — клиент MOEX ISS API с фоллбэком на sample_data
│   │   ├── parsers.py        — преобразование табличного формата MOEX ISS
│   │   ├── yield_calc.py      — расчёт доходностей и срока до погашения
│   │   └── filters.py        — фильтрация списков
│   ├── sample_data/         — тестовые данные в формате MOEX ISS (офлайн-фоллбэк)
│   ├── dev_server.py        — лёгкий сервер на стандартной библиотеке (без зависимостей)
│   ├── test_logic.py        — демонстрация работы логики без HTTP
│   └── requirements.txt
├── bot/
│   ├── bot.py               — Telegram-бот (aiogram), кнопка запуска мини-аппа
│   └── requirements.txt
├── frontend/
│   └── index.html           — мини-приложение (HTML/CSS/JS, Telegram WebApp SDK)
└── screenshots/             — скриншоты экранов (результат сквозной проверки)
```

## Правила приложения, заложенные в логику

- **Облигации**: показывается *текущая доходность* (к погашению), рассчитанная
  от актуальной цены — а не доходность за прошлый год. Если MOEX не отдаёт
  доходность к погашению (обычно облигации с переменным купоном), бэкенд
  считает приблизительную доходность по последней известной выплате и
  помечает `is_floating: true` — фронтенд показывает предупреждение, что
  доходность может измениться.
- **Акции**: дивидендная доходность считается *только по факту выплат за
  предыдущий календарный год* (`DIVIDEND_YEAR = текущий год − 1`, сейчас —
  2025) и явно подписана годом. На карточке акции также есть предупреждение,
  что дивиденды не гарантированы.
- **Интерфейс** — на русском, цветовая схема светло-сиреневый (`#EDE9FB`) +
  тёмно-синий (`#1B2A6B`), карточки/метрики/нижняя навигация в стиле
  банковских приложений.

## Быстрый старт

### 1. Бэкенд

Вариант А — полноценный, с FastAPI (рекомендуется):

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

Откроется автодокументация на `http://127.0.0.1:8000/docs`.

Вариант Б — без установки зависимостей (для быстрой проверки):

```bash
cd backend
python3 dev_server.py
```

Оба варианта реализуют один и тот же набор эндпоинтов:

| Метод | Путь | Описание |
|---|---|---|
| GET | `/api/meta` | год расчёта дивдоходности, список секторов |
| GET | `/api/bonds` | список облигаций (`yield_min/max`, `price_min/max`, `maturity`, `bond_type`) |
| GET | `/api/bonds/{secid}` | карточка облигации + график купонных выплат |
| GET | `/api/stocks` | список акций (`yield_min/max`, `price_min/max`, `sector`) |
| GET | `/api/stocks/{secid}` | карточка акции + история дивидендов |

**Реальные данные MOEX.** `moex_client.py` сначала пытается обратиться к
`https://iss.moex.com/iss/...` (бесплатный API без ключа). Если интернета
нет или MOEX недоступен — используются тестовые данные из `sample_data/`.
То есть на сервере с доступом в интернет приложение сразу заработает на
реальных котировках без дополнительной настройки.

**Список бумаг.** Сейчас `BOND_BOARDS` и `SECTOR_BY_SECID` в `data.py`
содержат небольшой демонстрационный набор. Чтобы расширить список —
добавьте нужные торговые доски/тикеры туда (MOEX ISS отдаёт сразу все
бумаги доски, дополнительно фильтровать руками не нужно).

### 2. Проверка логики без сервера

```bash
cd backend
python3 test_logic.py
```

Печатает таблицы облигаций/акций, пример фильтрации и карточки — на
тестовых данных, без сети и без FastAPI.

### 3. Фронтенд

Это статический HTML/JS, специальная сборка не требуется:

```bash
cd frontend
python3 -m http.server 8080
```

Откройте `http://127.0.0.1:8080/index.html` — по умолчанию фронтенд
обращается к API на `http://localhost:8000`. Чтобы указать другой адрес,
объявите перед загрузкой `index.html`:

```html
<script>window.DIVIBOND_API_BASE = "https://your-domain.example/api";</script>
```

Для Telegram Mini App потребуется HTTPS — задеплойте `frontend/` и
`backend/` на любой хостинг (например, как два сервиса за одним доменом).

### 4. Telegram-бот

```bash
cd bot
pip install -r requirements.txt
export BOT_TOKEN="токен_от_BotFather"
export WEBAPP_URL="https://your-domain.example/divibond/"
python3 bot.py
```

В @BotFather настройте Menu Button с тем же `WEBAPP_URL`.

## Скриншоты (сквозная проверка)

В `screenshots/` — результат автоматического прохода по приложению
(headless-браузер + dev-сервер на тестовых данных):

1. Главная, вкладка «Облигации»
2. Главная, вкладка «Акции»
3. Список облигаций

6. Список облигаций после фильтрации
7. Карточка акции SBER с историей дивидендов
8. Избранное (облигация + акция, удаление по корзине)

## Дальнейшие шаги

- Подключить кэширование MOEX-ответов (например, обновлять раз в 15 минут
  через `apscheduler`, отдавать из кэша).
- Расширить `BOND_BOARDS`/`SECTOR_BY_SECID` под полный список интересующих бумаг.
- Добавить экран «Настройки» (тема, уведомления об отсечках/купонах).
- Перенести избранное на сервер (сейчас — `localStorage`), используя
  `Telegram.WebApp.initDataUnsafe.user.id` как идентификатор пользователя.
